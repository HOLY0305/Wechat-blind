# 微信窗口隐私保护工具 — 需求文档

## 1. 项目概述

### 1.1 项目名称
WeChat Blind / 微信隐私幕布

### 1.2 设计原则
- **轻量级**：体积小（<5MB）、内存低（<10MB）、CPU 几乎无占用
- **无感运行**：系统托盘常驻，不干扰工作
- **即装即用**：像 Windows 小组件一样简洁，无需复杂配置

### 1.3 目标用户
开放式办公环境中需要保护微信聊天隐私的用户

## 2. 功能需求

### 2.1 核心功能（MVP）

#### FR-01: 智能遮罩
- 微信窗口失去焦点时自动显示磨砂遮罩
- 微信窗口获得焦点时自动隐藏遮罩
- 遮罩跟随微信窗口位置/大小实时同步
- 平滑过渡动画（200ms 以内）

#### FR-02: 系统托盘
- 最小化到托盘运行，无主窗口
- 托盘图标显示当前状态（启用/禁用）
- 右键菜单：启用/禁用、设置、退出

#### FR-03: 基础设置
- 开关遮罩功能
- 开机自启
- 快捷键切换（如 Ctrl+Shift+W）

### 2.2 辅助功能（v2）

#### FR-04: 高级设置
- 遮罩透明度调节
- 模糊程度调节
- 鼠标悬停策略（悬停时是否隐藏）

#### FR-05: 多显示器支持
- 正确处理跨屏场景
- 支持不同 DPI 缩放

## 3. 非功能需求

### 3.1 性能要求（严格）
| 指标 | 目标值 |
|------|--------|
| 安装包体积 | < 3MB |
| 内存占用 | < 10MB |
| CPU 占用 | < 0.1%（空闲时） |
| 启动时间 | < 1秒 |
| 焦点检测延迟 | < 50ms |

### 3.2 兼容性
- 操作系统：Windows 10/11
- 微信版本：PC 版（支持最新 3.x 版本）
- 架构：x64

### 3.3 稳定性
- 连续运行 7 天无崩溃
- 微信重启后自动重新检测
- 系统休眠/唤醒后正常工作

### 3.4 安全性
- 不读取/记录任何聊天内容
- 不发送任何网络请求
- 不修改微信进程或文件
- 本地运行，无云端依赖

## 4. 用户流程

### 4.1 安装启动
1. 运行安装包（或直接运行 exe）
2. 程序最小化到托盘，显示提示气泡
3. 检测到微信窗口后自动生效

### 4.2 日常使用
1. 用户正常使用微信
2. 当切换到其他窗口 → 遮罩自动出现
3. 点击微信窗口 → 遮罩自动消失
4. 通过托盘图标或快捷键临时禁用/启用

## 5. 约束条件

### 5.1 技术约束
- 使用 Windows 原生 API（Win32 API）
- 优先选择体积小、依赖少的技术栈
- 避免引入重型框架

### 5.2 用户体验约束
- 无弹窗打扰
- 无广告
- 无需注册/登录
- 无需联网

## 6. 已知问题

| 问题 | 严重程度 | 状态 | 说明 |
|------|----------|------|------|
| 遮罩显示在微信窗口下方 | CRITICAL | 待修复 | Z-order 管理有误，遮罩无法覆盖微信窗口 |
| 遮罩位置不同步 | HIGH | 待修复 | 微信窗口移动/缩放后遮罩不跟随 |
| DWM 模糊被 OnPaint 覆盖 | HIGH | 待修复 | OnPaint 绘制不透明背景，模糊效果无效 |
| TopMost 属性矛盾 | MEDIUM | 待修复 | 构造函数 TopMost=false 与 CreateParams WS_EX_TOPMOST 冲突 |
| WeChat 关闭后遮罩卡住 | MEDIUM | 待修复 | 无后台监控 WeChat 进程存活状态 |
| DPI 缩放偏移 | MEDIUM | 待修复 | GetWindowRect 返回物理像素，WinForms 用逻辑坐标 |
| 微信最小化时遮罩未隐藏 | LOW | 待修复 | 遮罩仍覆盖在最小化的微信上方 |
| 快捷键未实现 | HIGH | 未实现 | 需求要求 Ctrl+Shift+W 切换 |
| 设置面板未实现 | MEDIUM | 未实现 | 托盘菜单"设置"只弹提示 |
| 开机自启未实现 | MEDIUM | 未实现 | Settings 中有 AutoStart 字段但未接入 |
